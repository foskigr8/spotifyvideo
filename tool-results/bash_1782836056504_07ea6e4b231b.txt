The pipeline in plain terms:
Your already has every word with a start and end timestamp. The

playback engine reads the audio, and every ~3 seconds it fires a "draw this" event to an

strokes — paths, text, shapes — each with a time offset so they appear as the words are
spoken. The HTML Canvas API draws them stroke by stroke, not all at once, creating
the "being drawn in real time" effect.

challenge is that GLM needs to return precise coordinate paths, which requires either a
